## Notes

BUG-04 was a test-data issue, not a failure of the signup service. The service was correctly rejecting the weak password. The regression test now matches the current password rule and still verifies duplicate-account protection.

BUG-05 was caused by the Routine Builder only storing exercise names. The plus/minus controls changed the screen labels, but sets, reps, and rest were not saved in a transferable object or passed into the guided workout session. The fix created a detailed routine-selection path using `RoutineExerciseSelection`. The selected values are now stored, passed through the service layer, placed in `WorkoutSession`, and displayed by `UserGuidedWorkout`.
